# 企业员工管理系统 — 技术选型

## 一、整体架构

```
前端 (Vue 3 + Element Plus)  ⇄  后端 (FastAPI)  ⇄  数据库 (OpenGauss / PostgreSQL 协议)
```

---

## 二、后端技术栈

| 层次 | 技术 | 说明 |
|------|------|------|
| **Web 框架** | FastAPI | 高性能异步框架，自带 Swagger 文档，开发效率高 |
| **ORM** | SQLAlchemy 2.x | 成熟的 Python ORM，支持 PostgreSQL 协议，兼容 OpenGauss |
| **数据库驱动** | psycopg2-binary | PostgreSQL 官方 Python 驱动，OpenGauss 兼容 PG 协议可直接使用 |
| **数据校验** | Pydantic v2 | FastAPI 内置，用于请求/响应模型校验 |
| **服务器** | Uvicorn | ASGI 服务器，配合 FastAPI 使用 |
| **跨域处理** | FastAPI CORSMiddleware | 内置中间件，允许前端跨域访问 |

---

## 三、前端技术栈（可选，也可用纯命令行）

| 技术 | 说明 |
|------|------|
| **Vue 3** | 渐进式前端框架，轻量易上手 |
| **Element Plus** | 基于 Vue 3 的 UI 组件库，表格/表单/对话框开箱即用 |
| **Axios** | HTTP 请求库，与后端 API 交互 |
| **Vite** | 前端构建工具，开发热更新快 |

> 如果不需要 Web 页面，也可以只用 **命令行 (CLI)** 或 **Tkinter GUI** 作为前端。

## 八、E-R 模型核心实体

| 实体 | 主要属性 |
|------|----------|
| **员工 (Employee)** | 员工ID(PK)、姓名、性别、出生日期、手机号、邮箱、入职日期、薪资、部门ID(FK)、职位ID(FK) |
| **部门 (Department)** | 部门ID(PK)、部门名称、部门描述、创建时间 |
| **职位 (Position)** | 职位ID(PK)、职位名称、职位描述、薪资范围 |

**实体联系：**
- 部门 ⇄ 员工：一对多（一个部门有多个员工）
- 职位 ⇄ 员工：一对多（一个职位对应多个员工）
